# 员工创建高保真设计方案

本方案基于 `employee-creation-wireframes.html` 的 4 个核心页面继续细化，目标是给后续真实页面重构提供视觉、布局和交互基线。

## 设计定位

员工创建页是“数字员工主流程编辑器”，不是录制工具页。

视觉方向：

- 专业、克制、适合长时间使用的运营工具。
- 浅色控制台风格，避免营销页式装饰。
- 第一步定义工作以宽表单为中心，不强行展示主流程；第 2/3/4 步主任务流程占据页面视觉中心。
- 右侧只承载当前步骤的上下文操作。
- 调试阶段突出失败节点、Trace Preview 和运行历史。

## 页面结构

4 个核心页面严格对应左侧流程：

1. 定义工作
2. 录制主路径
3. 标注灵活节点
4. 试跑验证

统一布局：

```text
顶部：返回 / 员工名称 / 保存 / 试跑 / 发布
左侧：创建流程导航
第 1 步中间：定义工作表单
第 2/3/4 步中间：主任务流程
右侧：当前步骤上下文面板；第 1 步可不显示右侧面板
```

## 关键调整

- 录制阶段不再展示小型嵌入浏览器，默认打开独立录制窗口。
- `dry_run` 在界面上改成“安全试跑”，技术名只作为辅助标签。
- `Runtime` 和 `Row IDs` 进入高级调试设置，不占据默认操作区。
- “策略边界”不再作为独立窗口展示，改为智能节点配置里的安全约束说明。
- 试跑验证页以主流程运行状态、Trace Preview、Run Results 为主体。
